# Telegram bot: feature-stands manager

Бот на Python (`python-telegram-bot`) для управления фича-стендами, сгруппированными по микросервисам: роли, права доступа, inline-кнопки, хранение в JSON и ежедневные напоминания.

## Возможности

- **Роли**: Бэкенд / Фронтенд / Тестировщик / Наблюдатель (выбор при первом `/start`, смена через `/role`).
- **Права**:
  - **Бэкенд** — управляет только Backend-стендами.
  - **Фронтенд** — управляет только Frontend-стендами.
  - **Тестировщик** — управляет любыми стендами.
  - **Наблюдатель** — только просмотр статуса (без кнопок).
  - **Админ** (ID из `SUPER_ADMIN_ID`) — управляет любыми стендами, независимо от роли.
- **Статус**: 🟢 свободен / 🔴 занят (показывает `@username` и время захвата).
- **UI**: inline-кнопки, кнопка действия расположена сразу под строкой стенда, обновление через `edit_message_text`.
- **Напоминания**: ежедневно в 10:00 (МСК) бот напоминает пользователям о занятых ими стендах (JobQueue).

## Быстрый старт

### 1) Установить зависимости

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2) Настроить переменные окружения

Скопируйте пример и заполните значения:

```bash
cp .env.example .env
```

Переменные:

- **TELEGRAM_BOT_TOKEN** — токен бота
- **SUPER_ADMIN_ID** — ваш Telegram user id (целое число)
- **DATA_FILE** — путь к JSON (опционально, по умолчанию `data.json`)

### 3) Запустить

```bash
python bot.py
```

## Данные (JSON)

Файл `data.json` создаётся автоматически при первом запуске (если отсутствует) со стартовой структурой:

- `microservice_1`: `feature_stand_1..3` (backend)
- `frontend_app`: `feature_stand_1..2` (frontend)

## Команды

- `/start` — регистрация, выбор роли и приветствие
- `/role` — смена роли
- `/status` — актуальный список стендов и кнопки управления
