# Telegram bot: feature-stands manager

Бот на Python (`python-telegram-bot`) для управления фича-стендами, сгруппированными по микросервисам: роли, права доступа, inline-кнопки, хранение в **SQLite** и ежедневные напоминания.

## Возможности

- **Роли**: Бэкенд / Фронтенд / Тестировщик / Наблюдатель (выбор при первом `/start`, смена через `/role`).
- **Права**:
  - **Бэкенд** — управляет только Backend-стендами.
  - **Фронтенд** — управляет только Frontend-стендами.
  - **Тестировщик** — управляет любыми стендами.
  - **Наблюдатель** — только просмотр статуса (без кнопок).
  - **Админ** (ID из `SUPER_ADMIN_ID`) — управляет любыми стендами, независимо от роли.
- **Статус**: 🟢 свободен / 🔴 занят (показывает `@username` и время захвата).
- **UI**: inline-кнопки, кнопка действия расположена сразу под строкой стенда, обновление через `edit_message_text`.
- **Напоминания**: ежедневно в 10:00 (МСК) бот напоминает пользователям о занятых ими стендах (JobQueue).
- **SQLite**: данные пользователей/микросервисов/стендов/занятости хранятся в `bot.db` (по умолчанию).
- **Миграция**: при первом запуске бот может перенести данные из старого `data.json` (если БД ещё пустая).
- **Админка**: `/admin` (только для админов из переменных окружения) — добавление/удаление стендов через интерактивное меню.
  - можно **принудительно освободить** занятый стенд
  - можно удалить микросервис (при необходимости — с принудительным освобождением всех его стендов)

## Быстрый старт

### 1) Установить зависимости

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2) Настроить переменные окружения

Скопируйте пример и заполните значения:

```bash
cp .env.example .env
```

Переменные:

- **TELEGRAM_BOT_TOKEN** — токен бота
- **SUPER_ADMIN_ID** — Telegram user id админа (целое число)
- **SUPER_ADMIN_IDS** — список id админов через запятую (опционально)
- **DB_FILE** — путь к SQLite (опционально, по умолчанию `bot.db`)
- **DATA_FILE** — путь к старому JSON для миграции (опционально, по умолчанию `data.json`)

### 3) Запустить

```bash
python3 bot.py
```

## Данные (SQLite)

По умолчанию создаётся файл `bot.db`.

Если БД пустая, бот создаёт стартовую структуру:

- `microservice_1`: `feature_stand_1..3` (backend)
- `frontend_app`: `feature_stand_1..2` (frontend)

Если существует `data.json` и БД пустая, бот попробует выполнить **миграцию** из JSON в SQLite при старте.

## Команды

- `/start` — регистрация, выбор роли и приветствие
- `/role` — смена роли
- `/status` — актуальный список стендов и кнопки управления
- `/admin` — админка (добавить/удалить стенды)
